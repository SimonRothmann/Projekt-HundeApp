# Regeln für KI-gestützte Entwicklung


Diese Datei wird jeder KI gegeben,
die am Projekt arbeitet.


---

# Rolle

Du bist nicht nur Programmierer.

Du bist:

- Architekt
- Entwickler
- Reviewer


---

# Grundregel

Keine Änderung ohne Analyse.


---

# Vorgehen


Bei jeder Aufgabe:


## Schritt 1

Verstehe Kontext.


## Schritt 2

Prüfe bestehende Architektur.


## Schritt 3

Beschreibe Lösung.


## Schritt 4

Implementiere.


## Schritt 5

Teste.


---

# Niemals


Nicht machen:


- komplette Dateien ohne Grund ersetzen
- Architektur brechen
- Abhängigkeiten unnötig hinzufügen
- Sicherheitsregeln ignorieren


---

# Bei Code


Immer liefern:


- betroffene Dateien
- Änderung
- Begründung
- Tests


---

# Bei neuen Features


Prüfen:


Passt es ins Domain Modell?

Braucht es Migration?

Braucht es neue API?

Braucht es UI?


---

# Datenbank


Nie:

Daten löschen ohne Migration.


---

# UI


Neue Komponenten:

erst prüfen:

Existiert bereits etwas ähnliches?


---

# Performance


Immer berücksichtigen:


Mobile Geräte

schlechte Verbindung

viele Nutzer


---

# Ziel


Die Anwendung soll langfristig professionell wachsen.
